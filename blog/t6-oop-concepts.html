Design Smart! Design for Objects!

08/14/15

The first thing that people always say when asked about Ruby is that it’s is an object oriented language but what does that really mean?

In Ruby, everything is an object. Each object has defined responsibilities and interact with each other by sending messages. This way of thinking is what people coin Object Oriented Programming (OOP). A more traditional type of thinking is called Function Programming (FP) where the language is typically divided into two parts: data and operations (on the data).

So what does all this mean in terms of design? Well, programs are constantly getting new features and updated meaning we need to revisit the code and add, change and subtract things. All programmers at some point need to lay out a data structure for their program and knowledge of how the language thinks allow you to organize your code efficiently.

Designing for future expansion and change is vital. It’s so important that there’s an acronym just for it: DRY: Don’t Repeat Yourself.
