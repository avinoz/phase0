What's Hash & What's Array in Ruby

07/27/15

Ok, so what are these things hashes and arrays? Well, it’s storing and accessing data. Think of any program, in order for it to do anything, it needs to remember pieces of info and be able to retrieve them to do whatever you need it to do.

An array is a list. Think of a list of chores for example:

CHORES TODAY: 1) go to the bank 2) finish homework 3) lunch with Sophie

We give this array a name: chores_today[] . In this list we have three items but the crucial thing to know is that arrays start counting from 0 so ‘go to the bank’ is actually item number 0. We can call this item in the array by typing chores_today[0] . Similarly if we wanted to retrieve our lunch plans, we type chores_today[2]. How do we write this you ask? Well here it is:

chores_today = [ “go to the bank”, “finish homework”, “lunch with Sophie”]

As you can see, an array can hold anything really, integers, floats, strings.

Alright, so we know an array is a list, what’s hash? Think of hash being a list that you give to a friend to run your chores who needs to know more specific information like which bank? What homework?

chores_today = { "go to the bank" => "Wells Fargo", "finish homework" => "DevBootcamp", "lunch with Sophie" => "Mission Beach Cafe" }

To retrieve information from Arrays we use it’s position in the list. With hashes, we use the “key” instead. For example to retrieve “Wells Fargo” we type in chores_today[“go to the bank”] the key in this instant is “go to the bank”

Voila! Now you know, go give your girlfriend a shopping list in arrays and hashes!
